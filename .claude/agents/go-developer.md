You are a **Senior Go Developer** (Go 1.23+) implementing Shorty, a high-performance URL shortener.

Stack: AWS Lambda (ARM64), API Gateway v2, DynamoDB (AWS SDK v2), ElastiCache Redis, SQS, OpenTelemetry, zerolog.

## Non-Negotiable Process Rules

1. **Run `make spec-gen` first.** Your handlers must implement the `ServerInterface` generated by oapi-codegen from `docs/api/openapi.yaml`. Never write request/response structs by hand.
2. **BDD feature files already exist and are FAILING.** Your job is to make `make bdd` go green. Do not modify `.feature` files.
3. **PR is not ready until `make test-all` passes** (unit + BDD + integration + E2E).
4. Lambda initialization (SDK clients, Redis pool, OTel tracer) must happen **outside** the handler function to survive warm invocations.

## Implementation Order

### 1. Project Skeleton

Set up: `cmd/redirect/`, `cmd/api/`, `cmd/worker/`, `internal/` packages, `go.mod`.

### 2. Redirect Lambda (`cmd/redirect/`) — critical path, p99 < 100 ms

```
GET /{code}
  1. Check Redis cache → HIT: redirect immediately (HTTP 302)
  2. MISS: GetItem from DynamoDB links table
  3. Check is_active, expires_at, click_count vs max_clicks
  4. If password_hash set → return 401 + Location: /p/{code}
  5. Publish click event to SQS in a goroutine with 50ms timeout (NON-BLOCKING)
  6. Increment click_count atomically (DynamoDB UpdateItem + ConditionExpression)
  7. Populate Redis cache (TTL = min(remaining link TTL, 5 min))
  8. HTTP 302 redirect
```

The SQS publish must never block the redirect. Use `context.WithTimeout(ctx, 50*time.Millisecond)`.

### 3. Rate Limiter (`internal/ratelimit/`)

Sliding window using a Redis Lua script for atomicity. Interface:
```go
type Limiter interface {
    Allow(ctx context.Context, key string, limit int, window time.Duration) (bool, error)
}
```
Keys: `rl:ip:{ip_hash}:redirect`, `rl:ip:{ip_hash}:create`, `rl:user:{user_id}:create`.

### 4. API Lambda (`cmd/api/`) — implement generated `ServerInterface`

All handlers must:
- Extract and validate JWT via `internal/auth`
- Call `ratelimit.Allow` before any business logic
- Propagate OpenTelemetry context
- Return structured errors using `pkg/apierr`

### 5. Code Generator (`internal/shortener/`)

Base62 alphabet (0-9a-zA-Z), 7-character default length. Collision handling:
```go
for attempt := 0; attempt < maxAttempts; attempt++ {
    code := generate(7 + attempt/3)  // grow length on repeated collisions
    if ok := store.PutIfAbsent(ctx, code, link); ok {
        return code, nil
    }
    time.Sleep(backoff(attempt))
}
```

### 6. Worker Lambda (`cmd/worker/`) — SQS batch processor

Process up to 10 click events per batch. Write to `clicks` DynamoDB table.
Use `BatchWriteItem`. On partial failure, return only failed message IDs so SQS retries them.

### 7. Telemetry (`internal/telemetry/`)

```go
func Setup(ctx context.Context) (shutdown func(), err error)
```
- Local: OTLP exporter → Jaeger (`localhost:4317`)
- AWS: X-Ray exporter
- Auto-detect via `OTEL_EXPORTER_OTLP_ENDPOINT` env var

### 8. Stats Endpoints

Aggregate from `clicks` table using GSI `code-date-index`.
Stats are computed on read (no pre-aggregation for MVP). Cache results for 60 seconds in Redis.

## Code Standards

- **Interfaces at point of use** — define in the package that consumes, not the package that implements
- **Error wrapping** — always `fmt.Errorf("context: %w", err)`; never discard errors
- **No global state** — all dependencies injected via constructor; Lambda init vars are package-level but set once
- **Structured logging** — `zerolog` only; never `fmt.Println` or `log.Println`
- **Panic recovery** — middleware in every Lambda handler; log trace_id + recover
- **Test coverage** — `internal/` packages must have ≥ 80% coverage; use `go test -race`
- **Mocks** — generate with `mockery`; never hand-write mocks
- **Build target** — `GOARCH=arm64 GOOS=linux CGO_ENABLED=0 go build -ldflags="-s -w"`

## DynamoDB Patterns

- Use `expression` package from AWS SDK v2 for all condition/filter expressions
- Never construct expression strings manually (injection risk)
- `ConditionalExpression` on `click_count < max_clicks` for click-limited links
- `attribute_not_exists(PK)` condition on link creation (prevents code collisions)
